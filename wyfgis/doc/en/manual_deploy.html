Manual Deploy (Windows/Linux/...)
Last modified: 2010/8/1
Author: Wu Yongfeng
GIServer can be deployed manually，you need to install Java 1.5 and a Java web server.
Download Deploy Archive
Downlad deploy archive from http://sourceforge.net/projects/giserver/files/, the file name will be "giserver-[version]-deploy.zip", unzip it we will get "giserver.war" and samples, documents, etc.
Downlad and Install Java Runtime
Download Java Runtime(JRE/JDK 1.5) from http://www.oracle.com/technetwork/java/javase/downloads/index-jdk5-jsp-142662.html, install the release according to the instructions included with the it, set an environment variable JAVA_HOME to the pathname of the directory into which you installed the JRE/JDK.
In product enviroment you should install native JAI libraries for faster map render speed, they can be downloaded from:
JAI: https://jai.dev.java.net/binary-builds.html#Release_builds
JAI Image I/O: https://jai-imageio.dev.java.net/binary-builds.html#Release_builds
Download and install a Java Web Server
Such as Tomcat，notice to choose the version satisfied Java 1.5. Then, install and run it according to the instructions included with the it.
Deploy "giserver.war"
Deploy "giserver.war" to your Java web server.
Edit configuration file
Change directory to WEB-INF/conf, open file "server.properties", modify "WEB_ROOT" to your web server address (such as: http://localhost:8080), modify "MAP_SERVICE_DIR"to your map services directory (such as: C\:\\Program Files (x86)\\GIServer\\sample\\data\\mapservices , /home/giserver/data/mapservices, etc.).
Open file "administrator.properties", modify administrator accounts, only administrators will be allowed to login admin. console.
Start Up
Restart application "giserver" in Java web server.
Visit URL like "http://localhost:8080/giserver/" to ensure that GIServer has been deployed, if succeed you will see a page like this:

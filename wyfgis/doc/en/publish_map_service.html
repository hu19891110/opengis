Publish Map Service
Last modified: 2010/9/3
Author: Wu Yongfeng
GIServer use direcotries as map services, we can organize data, styles, configurations all below a directory, it will make updating and move services easily. In Config Tool, we can see map services directory is very important, GIServer will scan all directories and its sub folders for map services:
Prepare a Map Service Directory
First of all, create a new directory for a new map service, assuming we create a folder named "sample", and create a folder named "world" under it, it will be a map service named "sample:world" in GIServer.
Prepare Data and Styles
File data fource should be copied to map service directory, and we need SLD file for layers. Here we create a folder "data" to store Shapefiles, and create a folder named "sld" to store sld files.
（About SLD: http://www.opengeospatial.org/standards/sld）
We suggest using uDig to change layer's style, and export it as a SLD file. uDig is free, you can download from:http://udig.refractions.net/。
